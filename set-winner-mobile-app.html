Set Winner

App Design

Development of an app to view Tennis statistics and matches.

Tennis apps are limited and demonstrate usability issues and outdated designs. The goal was to design a structure that is both visually appealing to users and easy to navigate.
Areas
Research, Ideation, Visual Design, UI,UX.
Duration
4 weeks

Highlights

Goal
A tennis app that provides the user with the facility to view tennis results, find out the latest news and learn more about their favorite players.
User problem
The research found that one of the main problems for users was navigating the apps, as the content was quite difficult to find.
User problem
It was observed that users preferred to view the statistics in a browser rather than in the apps, due to the poor structure and lack of features in the apps.
Main challenge
How might we provide information about sport to the user in a way that is easy to understand and access?

Seeking opportunities

One of the first things I wanted to do was to understand the opinions of users on the different products that already try to solve the problem I am facing. Interviews, desk research and competitive analysis were conducted to find gaps in the experience.
Mostly negative comments about the apps' functions and usability.
Thanks to the competitive analysis, gaps were found on features and how content should be organised.
Final insights...
At the end of this phase, some insights were gained on what the app experience should be like.
In-depth statistics
Provide in-depth player and match statistics. Due to the interest of users to see every detail of the match.
Match feedback
Users want different ways to find out the results and actions of the match.
Simple organisation
Users do not want to browse an infinite number of screens to get to the element they want to see.

Users looking for a good structure

After researching the users and the market, some user personas were created to define our end user. After this, the main user problem to be solved was defined.
User journey
Because of the strong emphasis on having a usable experience, the User Journey was developed to refine the path that the user would take.
Problem statement
Users who play and watch tennis waste time on the product due to the poor structure and lack of in-depth statistical information. This generates frustration in the users, due to their limited time available for work. If the problem is solved, it would create user confidence, higher retention and satisfaction.
How might we...
Presenting the statistics and information of a match in a way that is easy to understand?
How might we...
Organise the games in a way that is intuitive?
How might we...
Make the shopping experience easy to follow?

Organizing the experience

Based on my research, I started to generate ideas in sketches that I then prioritized and placed in wireframes.
Task flows
Task flows were created for the different main user actions in the app.
Sitemap
A sitemap was defined in order to have an organised structure.
Wireframes
The creation of wireframes after the sketches helped to bring the ideas to the canvas.
Understanding Tennis has never been easier

Main features

With the help of user and market research, I added some important features to the application to make it more effective, have higher conversion and user satisfaction.
